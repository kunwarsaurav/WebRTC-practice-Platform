# AI Evaluation Engine - Backend Integration Guide

This document outlines how the Main Backend should integrate with the AI Evaluation Engine (hosted at `ml.synthbitietlsboost.com`).

Because the Frontend interacts directly with the AI Engine for WebRTC signaling and audio processing, the Main Backend's only responsibility regarding the AI is to **receive the final evaluation score** via a Webhook.

---

## 1. Environment Configuration

To enable the Webhook, you (the Main Backend team) must provide the AI team with an endpoint URL. The AI team will add this to their `.env` file.

**AI Engine Environment Variable:**
```env
MAIN_BACKEND_WEBHOOK_URL="https://your-main-backend.com/api/webhooks/ielts-score"
```

If this variable is set, the AI Engine will automatically trigger a `POST` request to this URL the moment an evaluation is complete.

---

## 2. Webhook Specification (What you need to build)

The Main Backend must expose a standard HTTP `POST` endpoint that expects a JSON body.

* **Method:** `POST`
* **Content-Type:** `application/json`

### Expected Payload
When a practice session concludes, the AI Engine will send the following JSON structure to your Webhook URL:

```json
{
  "roomId": "a1b2c3d4-5678-90ef-ghij-klmnopqrstuv",
  "userId": "usr_987654321",
  "report": {
    "fluency": 7.5,
    "lexical": 8.0,
    "grammar": 6.5,
    "pronunciation": 7.0,
    "overall": 7.5,
    "feedback": {
      "fluency": "Excellent job! You speak very naturally...",
      "lexical": "Great flow! You sound confident...",
      "grammar": "Good effort! You kept the conversation going...",
      "pronunciation": "Your speaking speed is fast and natural."
    }
  }
}
```

### Data Dictionary
| Field | Type | Description |
| :--- | :--- | :--- |
| `roomId` | `String (UUID)` | The unique ID for the practice room (generated by the AI API). |
| `userId` | `String` | The unique ID of the user (generated by *your* Main Backend when they logged in). Use this to attach the score to the correct user in your database. |
| `report.fluency` | `Float` | The IELTS band score for Fluency & Coherence. |
| `report.lexical` | `Float` | The IELTS band score for Lexical Resource. |
| `report.grammar` | `Float` | The IELTS band score for Grammatical Range & Accuracy. |
| `report.pronunciation` | `Float` | The IELTS band score for Pronunciation. |
| `report.overall` | `Float` | The overall calculated IELTS band score. |
| `report.feedback` | `Object` | Detailed string explanations for why the user received those specific scores, intended to be shown on the user's dashboard. |

### Expected Response from Main Backend
The AI Engine expects a `2xx` HTTP status code (e.g., `200 OK` or `201 Created`) confirming that you successfully received and saved the score.

```json
// Example success response
{
  "status": "success",
  "message": "Score saved successfully."
}
```

---

## Summary of the Data Flow
1. Main Backend authenticates the user and provides the Frontend with a `userId`.
2. Frontend requests a `roomId` from the AI Engine.
3. Frontend and AI Engine conduct the WebRTC session and Audio evaluation.
4. AI Engine fires this Webhook back to the Main Backend.
5. Main Backend saves the score to the database using the `userId`.
